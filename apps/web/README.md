# 成员网页端

成员网页端用于优先验证 MakersHub 的核心业务逻辑。

第一阶段重点覆盖登录、个人资料、积分流水、物资资源浏览、物资借用申请和我的借用记录。

项目立项、项目材料和复杂协会工作台体验不属于当前最小端到端验收主线；项目域进入 1.5 阶段。当前已有工作台任务页面可以保留，但后续开发应优先补齐物资借用主线，并把它从普通成员第一屏移入协会工作台。

## 长期定位

`apps/web` 长期应拆成公开主站首页 / 社区入口、成员中心和协会工作台三层，不是进站后直接显示登录框，也不是 BBS/论坛主站。

第一阶段可以做轻量公开主站首页，让未登录用户看到协会介绍、品牌视觉、登录/首次登录、小程序入口、6 个首页业务快捷入口和少量手工维护的内容预览；但不做完整 CMS、动态活动列表、动态项目展示、官方博客发布流或社区互动闭环。

登录后普通成员进入“成员中心”，不是进入协会工作台。成员中心优先展示资料状态、积分概览、我的待办、物资借用入口、我的借用记录摘要和常用入口。协会工作台入口只对协会工作人员或有权限用户显示，任务、值班、部门工作和简单审核放在协会工作台中。

后续可以逐步丰富为：

- 公开主站首页 / 社区入口：协会介绍、活动公告预览、项目/作品展示预览、社区内容预览、登录、小程序入口，以及个人借物、团队借物、场地申请、协会活动、项目立项、查看项目 6 个全局业务入口；
- 成员中心：我的资料、积分、物资借用、我的申请、我的项目、我的活动和常用入口；
- 协会工作台：任务、值班、扫除、远程控制、部门工作、申请和简单审核提醒；
- 资源：物资、场地和工位浏览与申请；
- 积分：余额、流水、规则和贡献记录；
- 项目：项目展示、立项、成员招募和材料；
- 内容：官方活动、公告和官方博客；
- 社区：讨论区、问答、作品分享和项目讨论。

其中社区可以做成 BBS-like 体验，但它是后续频道，不是当前主信息架构。第一阶段不要为了“像一个网站”而提前实现发帖、评论、举报、社区审核或社区积分。

## 共享 API 客户端

当前 `src/api.ts` 和 `src/auth-storage.ts` 只是成员网页端配置层：前者配置 `VITE_API_BASE_URL`，后者配置 `makershub.web.auth` 存储 key。
新版响应解析、`ApiRequestError`、令牌过期判断、`/auth/me`、`/permissions/me`、认证存储语义和工作台状态标签已经迁移到 `@makershub/api-client`。

成员网页端可以保留自己的页面流程、路由和 UI，但不应继续扩展一套只属于 `apps/web` 的 API 客户端。

## 当前已接入

- 邮箱密码登录；
- 首次邮箱验证码登录；
- 首次设置密码；
- access token 启动校验；
- refresh token 自动续签；
- 退出登录；
- 当前登录后任务验证页；
- 积分余额和最近流水；
- 工作台任务列表、可领取任务、任务发布、提交完成材料和发布人审核；
- 通过 `@makershub/api-client` 复用共享请求、认证、权限、错误和状态标签基础能力。

## 当前 P0 缺口

- 将当前偏任务验证的第一屏重排为公开主站、成员中心和协会工作台三层入口；
- 物资资源列表和详情；
- 物资借用申请；
- 修改待审核或已驳回物资借用申请；
- 我的物资借用记录；
- 与后台审核/归还联动的端到端验收。

## 第一阶段实现顺序

后续 Worker 拿任务时按以下顺序推进，不能跳过前置任务直接做静态页面：

1. 信息架构壳：匿名默认进入轻量公开主站首页；首页展示登录/首次登录、小程序入口和 6 个业务快捷入口，未接入入口标记后续能力。
2. 认证入口迁移：复用现有登录、首次登录、设置密码、会话恢复和退出逻辑；登录后进入成员中心。
3. 成员中心壳：把普通成员第一屏改为成员中心，展示资料状态、积分概览、物资借用入口和我的借用记录入口。
4. 协会工作台入口：只对协会工作人员或有权限用户展示；当前任务验证页移入协会工作台，不再占据成员中心。
5. 共享 API 补齐：先在 `@makershub/api-client` 补齐物资和借用 typed API，再接页面。
6. 物资浏览：接真实物资列表和详情，不使用 mock 数据冒充完成。
7. 借用申请：接真实申请接口，展示押金、可用积分、库存和资料完整性提示。
8. 我的借用记录：接真实列表/详情/修改/取消能力，不提供成员端归还入口。
9. 端到端联调：和后台审核/归还、积分流水、审计追溯一起验证。

本地开发默认连接：

```text
http://127.0.0.1:8000/api/v1
```

如需覆盖 API 地址，可设置：

```bash
VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1
```
