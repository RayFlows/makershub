# security

安全工具目录，负责密码哈希和访问令牌等底层安全能力。

本目录负责：

- 使用 bcrypt 生成和校验密码哈希；
- 签发和校验短期 access token；
- 将 JWT 过期、无效和类型错误转换为稳定业务错误码。
- 写入基础 HTTP 安全响应头；
- 限制普通接口请求体大小；
- 为认证接口和普通接口提供应用层兜底限流。

本目录不负责：

- refresh token 会话表；
- 用户登录业务流程；
- 权限点判断。
- 替代网关、WAF、Redis 限流或云厂商安全产品。

普通业务代码不能保存明文密码，也不能直接把微信 `openid` 当作系统用户主体写入 token。
生产多实例部署时，进程内限流只能作为兜底，仍应在网关或 Redis 层配置集中限流。
