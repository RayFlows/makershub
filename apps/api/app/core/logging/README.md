# logging

运行日志基础设施目录，负责 Loguru 初始化、标准 logging 桥接、文件分流和保留策略。

本目录负责：

- 控制台日志输出；
- `app.log`、`error.log`、`request.log`、`debug.log` 文件分流；
- 日志轮转、压缩和保留周期；
- 将 Uvicorn、SMTP 等标准库日志桥接到 Loguru；
- 在日志中贯穿 `request_id` 和日志分类。

本目录不负责：

- 业务审计；
- 积分流水；
- 长期集中日志存储。

运行日志用于排查服务异常和请求链路。审计日志用于回答“谁在什么时候改了什么”，两者不能互相替代。
