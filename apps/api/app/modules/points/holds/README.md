# 积分冻结能力

本目录负责冻结积分、解冻冻结记录，以及把冻结积分转为实际扣除。

冻结不是扣分，它只增加 `frozen_balance`，让可用余额减少。正常业务完成后解冻；逾期、
损坏、丢失或业务确认消耗时，才把冻结记录转为扣除。

## 当前能力

- `freeze_points(...)`：冻结可用积分并写入冻结流水；
- `release_point_hold(...)`：解冻有效冻结记录并写入解冻流水；
- `deduct_point_hold(...)`：把有效冻结记录转为实际扣除并写入扣除流水；
- 每一步都支持幂等键，避免业务重试造成重复冻结或重复扣除。

## 不负责什么

- 不决定借用、打印、任务等业务是否应该冻结；
- 不负责人工补发或扣减；
- 不负责 HTTP 权限和审计。
