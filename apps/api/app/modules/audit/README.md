# 审计

负责操作日志、审批轨迹、安全敏感事件和重要数据变更记录。

## 边界

审计不负责实际业务修改，也不替代积分流水、借用状态流水等业务事实。

- 运行日志：排查服务运行状态、异常和请求耗时；
- 审计日志：记录谁在什么时候对什么对象做了什么；
- 业务流水：记录积分、借用、审批等领域自己的事实流转。

## 当前已落地

- `audit_logs` ORM 模型；
- `AuditLogEntry` 和 `record_audit_log(...)` 服务；
- `GET /api/v1/audit/logs` 读取最近审计日志，需要 `system.audit.view`。

## 后续接入

以下写操作必须逐步接入审计：

- 绑定邮箱、设置密码、退出登录；
- 初始化或恢复 `999`；
- 修改成员部门、职务和状态；
- 人工积分调整；
- 资源新增、下架、库存调整；
- 借用审核、归还和异常处理。

审计记录默认只追加，不随意修改或删除。
