# 工作台

工作台负责协会内部日常运营，包括任务、悬赏、排班、值班、清洁任务和工作量统计。

特殊非模板任务必须先通过积分域的临时积分规则，再生成一次性任务模板；任务发布时只能
引用已有积分规则，不能临时填写积分数。

## 当前已落地

- `tasks`：任务发布、悬赏领取、完成提交、发布人审核、审核通过后按积分规则发放积分；
- 任务积分通过 `points.rules.grant_points_by_rule(...)` 进入积分账本；
- 任务完成产生的积分流水 ID 会回写到任务记录，便于后续对账。

## 当前目录结构

```text
workbench/
  models.py                     # 工作台任务模型
  constants.py                  # 任务状态、分配方式和可见范围
  types.py                      # 服务层分页结果
  utils.py                      # 文本和字段校验
  tasks/
    README.md
    service.py                  # 任务状态机和审核发分
    repository.py               # workbench_tasks 查询和写入
```

## 不负责什么

- 不负责积分规则维护，固定规则和临时规则在 `points.rules`；
- 不负责通知触达，后续由 notifications 域接入；
- 不负责内容发布和宣传链接审核，这些属于 content 域；
- 不负责项目生命周期，这些属于 projects 域。
