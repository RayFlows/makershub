# 物资借用

负责物资借用的申请、审批、取消和归还闭环。

## 已落地状态机

- `pending_review`：成员提交后待审核；
- `rejected`：管理员驳回，或审批时库存不足自动驳回；
- `approved`：审批通过，已扣减可借库存并冻结押金；
- `cancelled`：申请人取消，记录保留；
- `returned`：正常归还，恢复库存并解冻押金；
- `exception_closed`：损坏、遗失或消耗类归还，扣除押金，不自动恢复可借库存。

## 跨域依赖

- 资源域：读取和锁定 `materials`，审批通过扣减 `available_quantity`，正常归还恢复库存；
- 积分域：调用 `freeze_points`、`release_point_hold`、`deduct_point_hold` 处理押金；
- 审计域：HTTP 层记录申请、审核、取消和归还操作。

## 成员端动作边界

- “我的借用记录”展示本人全部申请状态，不只展示可操作记录；
- 成员只能修改或取消 `pending_review` 和 `rejected`，取消原因选填；取消已驳回申请只是用户侧收尾，不删除审核记录；
- 第一阶段个人物资借用必须填写预计归还时间，后端不能只依赖前端日期选择器；
- 修改申请沿用旧小程序 `edit=true&sb_id` 的业务语义：修改原申请，不要求取消后新建；
- 修改 `rejected` 后回到 `pending_review` 等待重新审核，并保留原审核记录；
- 申请记录保存申请人姓名、学号、手机号、联系邮箱、年级和专业快照；历史展示使用快照，权限和积分处理仍使用 `applicant_id`；
- 申请人快照从成员资料生成，借用表单不自由填写身份/联系字段；联系邮箱默认取登录邮箱且可在资料中修改，修改联系邮箱不等同于更换登录邮箱；联系邮箱必须由后端校验格式，但不需要验证码确认；姓名、学号、手机号、年级或专业缺失，或联系邮箱和登录邮箱均缺失时返回 `BORROW_PROFILE_INCOMPLETE`；
- 后台审核列表指后台管理端/后台主线审核控制台，不是普通成员网页端；
- 后台审核列表只展示摘要，不展示申请编号、学号、手机号、联系邮箱或当前联系方式；这些字段只在详情页展示；
- 后台审核列表不要拼接 `年级-专业-姓名`，应拆列展示姓名、年级、专业、提交时间、状态、类型、物资摘要和操作；
- 成员端“我的借用记录”列表是本人视角，不套用后台审核表格列；优先展示状态、提交时间、预计归还时间、物资摘要和可执行操作；
- 物资摘要使用借用明细快照生成：单项显示“物资名称 x 数量”；多项显示“首件物资名称 x 数量 等 N 项”，详情页再展示完整清单；
- 人工驳回物资借用申请必须填写驳回理由；审核通过意见选填，库存或积分不足导致的系统自动驳回由后端写入系统原因；
- 小程序是高频轻量入口，不承载物资台账维护、物资类别多层管理、库存调整、资源状态处理、复杂筛选导出和异常归还；待审核列表、通过/驳回和正常归还后续可以接入小程序管理端；
- 后台审核、归还和历史详情页主信息使用申请人快照；当前手机号或联系邮箱只能作为只读辅助联系信息，不改写申请快照、审核记录、归还记录或审计日志；
- 借用明细保存申请时物资名称、分类名称和单件押金快照；历史详情和审核展示使用快照，库存操作仍通过 `material_id` 访问当前物资台账；
- `approved` 表示已审核通过并等待后台归还处理，成员端不能提供归还或异常归还入口；
- `cancelled`、`returned` 和 `exception_closed` 在成员端只读展示。

## 旧系统差异

旧后端用字符串 `分类 - 名称 - 数量` 解析物资，并且取消会删除申请。新版使用
`material_id + quantity` 结构化明细，取消只变更状态，不删除历史记录。
