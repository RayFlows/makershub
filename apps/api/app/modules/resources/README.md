# 资源

负责物资、场地、工位、库存、资源状态和后续资源操作日志。

当前第一阶段已经落地物资分类、物资台账和库存快照：

- `models.py`：`resource_categories`、`materials`；
- `materials/`：物资分类、物资资料、库存调整服务；
- `constants.py`：资源类型、分类状态和物资状态常量；
- `types.py`：分页结果对象；
- `utils.py`：资源域文本和数量校验工具。

资源域不负责借用生命周期。借用申请、审批、取消、归还和押金冻结属于 `borrowing`。
借用域可以通过资源域仓储和服务在同一事务中修改库存，但不能把审批结论写回资源域。

物资台账维护、物资类别多层管理、库存调整和资源状态处理不放在小程序端。
小程序后续作为高频轻量入口，可以承接查看、提交、状态跟踪、简单审核和正常归还；
复杂物资管理能力放在浏览器端，管理类主入口为 `apps/admin`。
