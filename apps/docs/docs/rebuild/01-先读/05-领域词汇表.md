# MakersHub 领域词汇表

本文件统一重构期间的领域语言。后续 AI 或人工开发如果发现新词、歧义词或旧词冲突，
应优先更新这里，再改代码或页面文案。

## 端侧入口

| 术语 | 定义 | 不能混用 |
| --- | --- | --- |
| 成员网页端 | `apps/web`，浏览器中的成员入口，承载公开主站、成员中心和协会工作台入口。 | 不能叫后台端，也不能只按旧小程序能力设计。 |
| 成员中心 | 成员网页端中的普通成员自助区，用于资料、积分、物资浏览、借用申请和我的借用记录。 | 不能放审核归还、库存调整、系统管理。 |
| 协会工作台 | 成员网页端中面向协会工作人员的工作入口，承载任务、值班等协会内部工作。 | 不能等同于后台管理端。 |
| 后台管理端 | `apps/admin`，浏览器中的管理和审核入口，第一阶段只做主线审核控制台。 | 不能用静态菜单或角色名硬编码代替权限点。 |
| 小程序用户端 | 小程序中的普通成员高频入口，是成员网页端自助能力的轻量子集。 | 不能承载复杂台账维护。 |
| 小程序管理端 | 小程序中给管理人员使用的高频轻操作入口，是后台管理端部分能力的轻量子集。 | 第一阶段不做复杂异常归还和资源管理。 |

## 资源与借用

| 术语 | 定义 | 不能混用 |
| --- | --- | --- |
| 资源 | 可被借用或管理的协会资产抽象，例如物资、场地、工位。 | 不负责申请生命周期。 |
| 物资台账 | 物资基础资料、分类、位置、状态、总量、可借数量和押金积分。 | 不等于借用申请。 |
| 借用申请 | 成员为使用资源提交的业务事实，包含申请人快照、明细快照、理由、预计归还时间和状态。 | 不能把申请人资料改写成实时资料。 |
| 审核 | 有权限人员对待审核申请做通过或驳回。通过时扣库存并冻结押金。 | 不能用前端按钮禁用替代后端校验。 |
| 正常归还 | 物资完好归还，恢复库存并解冻押金。 | 不能和异常归还混成一个含糊按钮。 |
| 异常归还 | 损坏、遗失或消耗等风险归还，第一阶段全额扣除本次押金且不自动恢复库存。 | 第一阶段不做部分扣除或普通撤销。 |

## 状态与验收

| 术语 | 定义 | 不能混用 |
| --- | --- | --- |
| 后端能力已具备 | 模型、服务、接口或测试已具备，但端侧尚未完整接入。 | 不能写成可验收完成。 |
| 端侧已接入 | 页面或小程序调用真实接口。 | 不能使用 mock 数据或静态按钮冒充。 |
| 可本地演示 | 本地环境可按账号和步骤走通主流程。 | 仍可能有测试或异常分支缺口。 |
| 可验收完成 | 满足端到端主线、权限、状态流转、积分/审计追溯、测试和文档。 | 不能只看后端文件数量或页面数量。 |

## 维护要求

- 端侧能力按包含关系理解：小程序用户端是成员中心高频子集，小程序管理端是后台管理端高频子集。
- 所有判断层面的业务规则必须由后端兜底，前端只做体验优化。
- 业务术语变化必须同步更新需求清单、路线图、API 契约或本词汇表。
